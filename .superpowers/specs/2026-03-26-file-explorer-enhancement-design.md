# ファイルエクスプローラー強化 設計書

## 概要

MDiumのファイルエクスプローラーを、Windows Explorerのような本格的なファイル管理機能を備えたものに強化する。

## 実装アプローチ

フロントエンド中心のアプローチを採用する。ドラッグ&ドロップやクリップボード操作はReact側で管理し、TauriバックエンドにはファイルI/Oコマンドのみ追加する。既存のアーキテクチャ（React + Zustand + Tauriコマンド）に自然に沿い、追加の依存を最小限に抑える。

---

## セクション1: 全表示モードとフィルタの変更

### 現状

- `.md`ファイルは常に表示
- 他のファイルタイプ（画像、docx、xls、km、pdf）はフィルタボタンでトグル追加表示
- フィルタOFF時は`.md`のみ表示され、全ファイルは見えない

### 変更内容

- **デフォルトを「すべて表示」に変更**: 全ファイル・全フォルダが表示される
- **フィルタは「絞り込み」として機能**: 選択したタイプのみに絞り込む
- **フィルタは複数選択可能**: .md + 画像など、複数タイプを同時に表示できる
- **「すべて」ボタンを追加**: 個別フィルタをすべて解除して全表示に戻る

### フィルタ操作ルール

- 「すべて」ON → 全ファイル表示（個別フィルタはすべてOFF）
- 個別フィルタをクリック → 「すべて」がOFFになり、選択したタイプのみ表示
- 複数の個別フィルタを同時ON可能（.md + 画像 など）
- 個別フィルタをすべてOFF → 自動的に「すべて」がONに戻る
- 「すべて」をクリック → 個別フィルタをすべて解除して全表示に戻る

### バックエンド変更

- `get_file_tree`: `filters`が空の場合は全ファイルを返却（隠しファイル・node_modules等の除外は維持）
- `include_empty_dirs`: 全表示モード時は常にtrue

## セクション2: ファイル/フォルダ操作（移動・コピー・削除）

### コンテキストメニュー（右クリック）

変更後のメニュー項目:

1. 📂 新しいタブで開く（フォルダのみ）
2. 🌐 外部アプリで開く（新規追加）
3. ─ セパレータ ─
4. ✂️ 切り取り (Ctrl+X)
5. 📄 コピー (Ctrl+C)
6. 📋 貼り付け (Ctrl+V)
7. ─ セパレータ ─
8. ✏️ 名前変更 (F2)（既存）
9. 🗑️ 削除 (Del)（既存）

**削除項目:** 「名前をコピー」は削除する。

### キーボードショートカット

| ショートカット | 動作 |
|--------------|------|
| Ctrl+X | 選択中のファイル/フォルダを切り取り |
| Ctrl+C | 選択中のファイル/フォルダをコピー |
| Ctrl+V | 選択中のフォルダ内に貼り付け（ファイル選択時はその親フォルダに貼り付け） |
| Delete | 削除（確認ダイアログ付き、既存） |
| F2 | 名前変更（既存） |

### クリップボード管理

Zustand storeに`clipboardEntry`を追加:

```typescript
clipboardEntry: {
  path: string;       // 対象ファイル/フォルダのパス
  mode: 'cut' | 'copy';
} | null
```

- 切り取り時: ツリー内でそのノードを半透明表示（視覚フィードバック）
- 貼り付け後: 切り取りの場合は移動実行、コピーの場合は複製実行
- 同名ファイルが存在する場合: 確認ダイアログを表示
- 貼り付け後にツリーを自動リフレッシュ

### バックエンド追加コマンド

- `copy_file(src: string, dest: string)` — ファイル/フォルダを再帰的にコピー
- `move_file(src: string, dest: string)` — ファイル/フォルダを移動
- `open_in_default_app(path: string)` — OS関連付けアプリで開く

## セクション3: ドラッグ&ドロップ

### 3-A: ツリー内ドラッグ&ドロップ

**操作:**

- ファイル/フォルダをドラッグしてフォルダ上にドロップ → 移動（デフォルト）
- Ctrlキーを押しながらドロップ → コピー
- ドラッグ中のカーソル: 移動時は通常カーソル、コピー時は「+」マーク

**ビジュアルフィードバック:**

- ドラッグ元: 半透明表示（opacity: 0.4）
- ドロップ先フォルダ: 背景ハイライト + 破線ボーダー
- ドロップ不可の場所（自分自身や子フォルダ）: ドロップ禁止カーソル

**制約:**

- フォルダを自分の子孫フォルダに移動不可
- 同名ファイルが存在する場合は確認ダイアログ表示
- ルートフォルダの外には移動不可

**実装方式:** HTML5 Drag and Drop API

```
onDragStart → ドラッグ元パスをdataTransferにセット
onDragOver  → ドロップ先判定、ハイライト表示
onDrop      → move_file or copy_file 実行
```

### 3-B: OS → ツリーへのドロップ

**仕組み:**

- Tauriの`onDragDropEvent`でドロップイベントを検知
- ドロップ位置からツリー内のフォルダを判定（マウス座標 → DOM要素 → パス）
- フォルダ上にドロップ → そのフォルダ内にコピー
- ファイル上にドロップ → そのファイルの親フォルダにコピー
- ツリーの空白領域にドロップ → ルートフォルダにコピー

**既存の画像D&Dとの共存:**

- エディタ領域へのドロップ → 既存動作維持（マークダウン画像挿入）
- ツリー領域へのドロップ → 新機能（ファイルコピー）

### スコープ外

- ツリー→OS外へのドラッグ（Tauriの制約上、将来対応とする）
- 複数ファイルの同時ドラッグ（複数選択が未実装のため）

## セクション4: データフローとエラーハンドリング

### 状態管理（Zustand Store 拡張）

file-store.tsに以下を追加:

```typescript
clipboardEntry: {
  path: string;
  mode: 'cut' | 'copy';
} | null;

dragState: {
  sourcePath: string;
  dropTargetPath: string | null;
} | null;

// アクション
setClipboard(path: string, mode: 'cut' | 'copy'): void;
clearClipboard(): void;
setDragState(source: string, target: string | null): void;
clearDragState(): void;
```

### ファイル操作後の整合性

**移動/名前変更 時:**

- 対象ファイルが開いているタブ → タブのパスを新パスに更新
- 対象フォルダ配下のファイルが開いているタブ → 全タブのパスを更新
- アクティブファイルが対象 → アクティブファイルのパスも更新

**削除 時:**

- 対象ファイルが開いているタブ → タブを閉じる
- 対象フォルダ配下のファイルが開いているタブ → 全タブを閉じる

**すべての操作後:**

- ファイルツリーを自動リフレッシュ

### エラーハンドリング

| エラーケース | 対応 |
|------------|------|
| 同名ファイル/フォルダが存在 | 確認ダイアログ「上書きしますか？」（上書き/スキップ/キャンセル） |
| 自分の子孫にドロップ | ドロップ禁止カーソル表示、操作を無視 |
| ファイルがロック/使用中 | エラートースト通知を表示、操作中止、ツリーをリフレッシュ |
| パスが長すぎる / 不正な文字 | バックエンドでバリデーション、エラーメッセージを返却 |
